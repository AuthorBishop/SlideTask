# 需求文档

## 1. 应用概述

### 1.1 应用名称
多任务管理App

### 1.2 应用描述
一款极简风格的移动端任务管理应用，通过可视化进度条与节点形式帮助用户管理多个任务的执行进度。用户可直观查看任务当前状态，通过拖动进度条快速更新任务进度，界面采用圆角柔和设计，提供舒适放松的使用体验。

## 2. 用户与使用场景

### 2.1 目标用户
需要同时管理多个任务或项目的个人用户，包括但不限于：
- 项目管理者
- 学生（管理学习任务）
- 自由职业者
- 需要跟踪多项工作进度的职场人士

### 2.2 核心使用场景
- 用户创建新任务并设置任务节点
- 用户在首页快速查看所有任务的当前进度
- 用户完成某个节点后，拖动进度条更新任务进度
- 用户在首页直接编辑任务节点描述
- 用户查看任务详情并管理节点信息

## 3. 页面结构与功能说明

### 3.1 页面结构

```
多任务管理App
├── 首页（任务列表页）
│   ├── 任务组件（可展开/收起）
│   └── 新建任务入口
└── 任务详情页
    ├── 任务基本信息编辑
    └── 节点管理
```

### 3.2 首页（任务列表页）

#### 3.2.1 页面布局
- 顶部：应用标题或Logo
- 中部：任务列表区域，纵向滚动展示所有任务
- 底部或右下角：新建任务按钮（圆角设计）

#### 3.2.2 任务组件
每个任务以独立组件形式展示，包含以下元素：
- 任务名称/标识：位于进度条上方或左侧
- 进度条：从左到右的填色条，填色部分表示已完成进度
- 节点标记：进度条上标记多个节点位置，每个节点对应一个子任务
- 节点文字提示：进度条上方显示当前填色点前一个节点和下一个节点的描述文字，当同一行空间不足以并排显示多个节点的完整文字时，采用上下错开布局策略，将节点在垂直方向上错位排列，确保所有节点文字完全显示，不被截断或隐藏
- 颜色或图标：可选，用于区分任务状态或优先级

#### 3.2.3 进度条交互
- 用户点击进度条，展开编辑菜单（带平滑过渡动画）
- 编辑菜单包含：
  - 拖动滑块：用户可拖动填色条至目标节点与下一节点中间位置，更新任务进度，拖动进度条仅调整进度值，不触发或改变任务节点列表的排序
  - 编辑节点描述：点击节点文字可直接编辑当前节点或下一节点的描述
  - 查看详情：进入任务详情页
- 用户完成拖动或编辑后，菜单自动收起（带平滑过渡动画）

#### 3.2.4 新建任务
- 用户点击新建任务按钮
- 弹出新建任务表单（带平滑过渡动画），包含：
  - 任务名称输入框
  - 节点设置：用户可添加多个节点，每个节点包含节点描述
  - 确认按钮与取消按钮（圆角设计）
- 用户填写完成后点击确认，任务添加至首页列表顶部

### 3.3 任务详情页

#### 3.3.1 页面布局
- 顶部：返回按钮、任务名称
- 中部：任务详细信息区域
- 底部：删除任务按钮（圆角设计）

#### 3.3.2 任务基本信息编辑
- 任务名称：可编辑
- 任务备注：可选，用户可添加任务相关说明

#### 3.3.3 节点管理
- 节点列表：展示所有节点及其描述
- 节点编辑：用户可修改节点描述
- 节点添加：用户可在任意位置插入新节点
- 节点删除：用户可删除节点
- 节点排序：用户可长按节点并拖动调整节点顺序，重排序操作实时生效并同步更新数据库

#### 3.3.4 删除任务
- 用户点击删除任务按钮
- 弹出确认提示框（带平滑过渡动画）
- 用户确认后，任务从列表中移除，返回首页

## 4. 业务规则与逻辑

### 4.1 进度条填色规则
- 进度条填色范围：从左侧起点到用户拖动的位置
- 填色位置必须在某个节点与下一节点之间，表示当前节点已完成，下一节点尚未完成
- 填色位置不能超过最后一个节点

### 4.2 节点状态判断
- 已完成节点：填色条已覆盖的节点
- 当前节点：填色点前一个节点（最近完成的节点）
- 下一节点：填色点后第一个未填色的节点（待执行的节点）
- 未开始节点：下一节点之后的所有节点

### 4.3 节点文字显示规则
- 进度条上方显示当前节点和下一节点的描述
- 每个节点的文字标签必须完全显示，不可被截断或隐藏
- 当同一行空间不足以并排显示多个节点的完整文字时，采用上下错开布局策略，将节点在垂直方向上错位排列，保证所有文字内容清晰可见
- 若任务刚创建且进度为0，显示第一个节点描述
- 若任务已完成所有节点，显示最后一个节点描述

### 4.4 节点排序规则
- 节点顺序的变更仅能通过任务详情页的长按拖拽操作触发
- 拖动进度条调整进度值不触发或改变任务节点列表的排序
- 节点重排序操作实时生效并同步更新数据库

### 4.5 任务数据存储
- 所有任务数据需持久化存储在本地
- 任务数据包括：任务名称、节点列表（节点描述、节点顺序）、当前进度位置、任务备注、创建时间

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户创建任务时未填写任务名称 | 提示用户「任务名称不能为空」，禁止创建 |
| 用户创建任务时未添加任何节点 | 提示用户「至少需要添加一个节点」，禁止创建 |
| 用户删除任务时误操作 | 弹出确认提示框，用户需二次确认才能删除 |
| 用户在详情页删除所有节点 | 提示用户「至少需要保留一个节点」，禁止删除 |
| 用户拖动进度条至节点位置 | 自动吸附至节点与下一节点中间位置 |
| 任务列表为空 | 显示空状态提示「暂无任务，点击新建任务开始管理」 |
| 用户长按拖拽节点时误触其他操作 | 长按拖拽状态下禁用其他交互，直至拖拽完成或取消 |

## 6. 视觉与体验设计要求

### 6.1 整体风格
- 界面整体感觉舒适放松，避免过于严肃或紧张的视觉氛围
- 配色柔和，避免高饱和度或强对比色

### 6.2 字体选择
- 避免过于方正的字体
- 建议使用柔和圆润或具有人文感的无衬线字体
- 字体大小层级清晰，确保可读性

### 6.3 圆角设计
- 所有UI组件（按钮、进度条、卡片、输入框、弹窗等）均采用圆角设计
- 圆角弧度需统一和谐，建议使用相同或成比例的圆角半径

### 6.4 过渡动画
- 所有UI状态切换（展开/收起、编辑/预览、弹窗显示/隐藏、页面跳转等）需带有平滑的过渡动画
- 动画时长建议在200-400ms之间，避免过快或过慢
- 动画曲线使用缓动函数，避免线性变化

### 6.5 进度条视觉
- 进度条背景色与填色部分需有明显区分
- 节点标记清晰可见，建议使用圆点或小图标
- 填色部分可使用渐变色或纯色，确保视觉舒适

### 6.6 节点拖拽视觉反馈
- 用户长按节点时，节点需提供视觉反馈（如轻微放大、阴影增强、透明度变化等）
- 拖拽过程中，被拖动的节点需与其他节点有明显区分
- 拖拽目标位置需有清晰的视觉提示（如插入线、占位符等）

## 7. 验收标准

1. 用户打开App，进入首页，查看任务列表（若为空则显示空状态提示）
2. 用户点击新建任务按钮，填写任务名称并添加至少2个节点，点击确认，任务成功创建并显示在首页列表顶部
3. 用户在首页点击某个任务的进度条，展开编辑菜单，拖动填色条至第一个节点与第二个节点中间位置，进度条填色更新，节点文字显示当前节点和下一节点描述，且文字完整显示不被截断
4. 用户点击任务进度条上方的节点文字，直接编辑节点描述，保存后文字更新
5. 用户点击编辑菜单中的「查看详情」，进入任务详情页，长按某个节点并拖动至新位置，节点顺序实时更新，返回首页后节点顺序保持一致
6. 用户在任务详情页点击删除任务按钮，确认删除后，任务从首页列表中移除
